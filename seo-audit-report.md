# YobuHo SEO総合診断レポート

> 診断日: 2026-05-10
> 対象: yobuho.com（呼べるホテル検索ポータル）
> 規模: 中規模（公開URL ~600件、サブディレクトリ展開含む）
> 担当: seo-lead（4専門領域を統合診断）

---

## 0. エグゼクティブサマリー

YobuHoはSEOの土台（サイトマップ・構造化データ・モード別SSG・パンくず・Cloudflare CDN・GSC連携）が中規模サイトとしては**かなり丁寧に整備**されています。インデックス可能ページ数（348+）、ガイドページ4本、Astro SSG化、サブディレクトリ方式URL、JSON-LD（WebApplication / BreadcrumbList / Organization）、モード別 H1/canonical/title すべて整っており、これは10年級のサイトでも到達できていない水準です。

ただし**重大な穴が3つ**あり、これらを潰さないと「土台が整っているのに伸びない」状態が続きます:

1. **ガイドページのCTAリンクが完全に壊れている**（プロトコル相対URLの誤記）→ せっかくのロングテール流入が回遊しない
2. **サブドメイン（deli.yobuho.com 等）に同じLPが配信されているのに canonical はパス側を指す** → 重複コンテンツ判定リスク + サブドメインの被リンク資産が活きない
3. **「ラブホ 口コミ / ラブホ 一人入室」という主要KWの専用LPが存在しない**（loveho LPのcanonicalが `?tab=loveho` という不安定URLを指している）→ ラブホ系で順位が伸びない構造

これら3点を直すだけで、CTR・回遊・インデックス被覆率が体感レベルで改善する見込みです。

## 1. 4領域別 診断結果

### テクニカルSEO（seo-technical観点）

#### 良い点
- **CSP / セキュリティヘッダー**: HSTS / X-Content-Type-Options / Referrer-Policy 完備、CSPで外部ドメインホワイトリスト
- **Cloudflare CDN + 自動Purge**: デプロイ後にAPI経由でPurgeしている。多くのサイトはここまで自動化していない
- **Astro SSG化**: portal-{mode}.html / ガイドページ / サブドメインLPが**ビルド時にtitle/description/canonical/H1確定** → JS依存ゼロでGooglebotがクロールしてすぐにメタ情報取得可能
- **CLAUDE.md記載のH1上書き禁止ルール**を ui-utils.js が遵守: `isSSG` 判定で SSG ページではメタを上書きしない設計（feedback_check_claudemd_decisions ルール準拠）
- **構造化データ**: WebApplication / SearchAction / BreadcrumbList / Organization / Article（ガイド）まで実装
- **sitemap.xml**: 348+ URL、daily/weekly のchangefreq適切、URLエンコードで&エスケープ済み
- **robots.txt**: admin / shop-admin / api / sql / scripts / *.bak を Disallow、ノイズURL除外済み
- **GSC + Bing 登録済み**: ドメインプロパティ + サイトマップ送信済み

#### 問題点・改善余地

| 影響度 | 容易度 | 項目 | 内容 |
|---|---|---|---|
| HIGH | 易 | サブドメイン重複コンテンツ | deli.yobuho.com / jofu.yobuho.com / same.yobuho.com / loveho.yobuho.com / este.yobuho.com の5つに同じAstro LPが配信されているが、canonical はすべて https://yobuho.com/deli/ 等のパス側を指している。サブドメインから見るとcanonicalがクロスドメイン → Googleは「同一サイトの正規URLを指している」とは認識しづらい。**結論**: サブドメインを生かすか（=独自canonical化）、潰すか（=サブドメインから301リダイレクト）の方針決定が必要。現状は「重複コンテンツがゾンビ化」した状態 |
| HIGH | 中 | ラブホ系LPの canonical設計 | astro-src/src/pages/loveho/index.astro の canonical が https://yobuho.com/deli/?tab=loveho という**JSが必要なクエリ付きURL**を指している。SSG化しているのに、canonical先がSPA挙動依存 → Googleが「正規URL=動的ページ」と認識しソフト404扱いリスク。「ラブホ 口コミ」「ラブホ 一人入室」という主要KW専用の独立LP（/loveho/ 等）を作るべき |
| MEDIUM | 易 | ガイドページの sitemap URL | sitemap.xml はガイドページを /guide/deli-hotel.html で登録しているが、AstroのデフォルトURL形式かつクリーンURL（/guide/deli-hotel/）への301リダイレクトがない。.html 付きURLがインデックスされ続ける |
| MEDIUM | 易 | Pagefind WASMのCSP | CSPに 'wasm-unsafe-eval' を追加済みだが、Pagefindのインデックス取得もconnect-srcで 'self' のみ（OK）。問題は**初回ロード33KB+51KB**の負荷。LCPに影響する可能性 |
| MEDIUM | 中 | フォント読み込み | Cormorant Garamond を全モードで読み込み。ロゴ装飾用だが、display=swap でCLS発生中の可能性。font-display:optional か preload + display=swap を検証 |
| LOW | 易 | hreflang | 多言語化（ja/en/zh/ko）を i18n でクライアント側翻訳しているが、hreflang を出していない。現状は日本語のみインデックスでOKだが、将来的に英語翻訳ページを別URL化するなら必要 |
| LOW | 中 | ホテル詳細ページのSSR/SSG化 | /deli/hotel/12345 クリーンURLは設定済みだが、portal-seo.php がH1を動的書き換えしている（CLAUDE.md記載）。43,580件のホテル詳細ページが個別SEOキーワードを持てる状態か再点検が必要 |

### コンテンツSEO（seo-content観点）

#### 良い点
- **モード別title/description**: 5モード × 国内47都道府県 × 主要市区町村のメタ情報がSSGで個別生成
- **ガイドページ4本**: ロングテール「デリヘル ホテル 選び方 / 女風 ホテル / LGBT ホテル / デリエステ ホテル」を狙ったArticle構造化データ付きの解説記事
- **キーワード密度**: H1・description・JSON-LD・フッターSEOテキストすべてに主要KWが入っており、過剰でも不足でもない
- **JoFu = 女風 / Deli = デリヘル / Same = 同性利用 / Este = デリエステ** とジャンル別ブランド名を切り分け、SEO的にもE-E-A-Tにも有利
- **MODE_HERO_H1_MAP**: SPA遷移時にH1がモード別文言に戻る（resetGenreHeroText） → SEO H1リセット問題（feedback_check_claudemd_decisions）対策済み

#### 問題点・改善余地

| 影響度 | 容易度 | 項目 | 内容 |
|---|---|---|---|
| **CRITICAL** | 極易 | **ガイドページのCTAリンクが壊れている** | astro-src/src/pages/guide/deli-hotel.astro:115,128 / jofu-hotel.astro:109,118 / lgbt-hotel.astro:114,115,124 の合計 **7箇所**で `<a href="//deli/">` というプロトコル相対URL記法。これはブラウザが https://deli/ として解釈してしまうため、**ガイド記事から検索ページへの誘導が完全に死んでいる**。せっかくのロングテール流入が CV に繋がらない最大級の漏れ |
| HIGH | 中 | ラブホ系コンテンツの不足 | 「ラブホ 口コミ」「ラブホ 一人入室」「ラブホ 査定」は CLAUDE-SEO.md でも主要KWだが、専用LPが存在せず、/deli/?tab=loveho というクエリ付きURLしかない。検索意図「ラブホテルの口コミを見たい」と「デリヘルホテルを探したい」は別物のため、/loveho/ 独立パス + 独立LP化が必要 |
| HIGH | 中 | 都道府県 × 市区町村ページの「中身」 | sitemap には /deli/東京都/新宿区 等の348+ URLが入っているが、これらは Astro SSG ではなく portal-{mode}.php で動的レンダリングされる（H1は「デリヘルを呼べるホテル検索」のままで都市名が入らない可能性）。**都道府県・市区町村名が H1 / title / description に動的に入っているか要検証**（入っていなければ全部同一コンテンツ扱い） |
| MEDIUM | 中 | ガイドページの数 | 4本（デリヘル / 女風 / LGBT / デリエステ）はあるが、検索意図カバレッジが薄い。「カードキー ホテル デリヘル」「フロント相談 やり方」「直通エレベーター ホテル 一覧」「ラブホ 一人 安全」「ラブホ 査定 落ちる理由」など、サジェスト系トピック10本程度に拡張する余地あり |
| MEDIUM | 中 | ホテル詳細ページの口コミ密度 | 43,580件のホテルがあるが、口コミが0件のホテルが大半。Googleが「内容が薄い」と判断するとインデックス被覆率が下がる。**口コミ0件のホテルは noindex、5件以上は index** の自動切り替えロジックが有効 |
| LOW | 易 | フッターSEOテキストの差別化 | 全モードのフッターSEOテキストがほぼ同じ文言（"YobuHoは全国43,000件以上..."）。モード別に固有の文言を入れると重複コンテンツ判定の保険になる |

### サイト構造（seo-structure観点）

#### 良い点
- **サブディレクトリ方式URL**: /deli/東京都/東京２３区内/渋谷区 のような階層が浅く理解しやすい構造
- **クリーンURL**: /deli/hotel/12345 / /deli/shop/my-slug/ がパスベース化済み（旧クエリURLは301リダイレクト）
- **パンくずリスト**: BreadcrumbList JSON-LD + 視覚的パンくず両方完備
- **内部リンク網**: フッターから全モード + 法的ページ + 店舗登録に動線あり、モードドロップダウンで他ジャンルに即遷移可能
- **モード切替UI**: portal-header から他ジャンルに1クリックで移動できる（クロスモード回遊）
- **H1ヒーローセクション**: モード別 H1 がページ上部に明示されている

#### 問題点・改善余地

| 影響度 | 容易度 | 項目 | 内容 |
|---|---|---|---|
| HIGH | 易 | ガイドページ → 検索ページの動線断絶 | 上記CRITICAL（壊れた //deli/ リンク）と同義。せっかくPageRankが流れてくるガイドから、検索ページへの内部リンクが死んでいる |
| HIGH | 中 | サブドメインの位置づけが曖昧 | deli.yobuho.com が「LPだけ」「LPからパス側に飛ばす」設計だが、サブドメインのフッターはパスURLを指している → サブドメインに来た人は1クリックでパスURLに飛ぶが、被リンクをサブドメインで受けた場合のSEOパワーが分散される。**「サブドメインを正規にする」か「301でパスに統合する」**の方針決定が必要（CLAUDE-SEO.mdでは "deli.yobuho.com 等" と記載されているのでサブドメインは生きている前提） |
| MEDIUM | 中 | ガイド ↔ 都道府県ページ の相互リンク不足 | ガイド「デリヘル ホテル 選び方」から「東京都/新宿区」のような実際の地域ページへの誘導がCTA1個だけ（さらにそれが壊れている）。「東京・大阪・名古屋・福岡」等の主要都市リンクをガイドに置くと、ロングテール流入が地域ページに分散できる |
| MEDIUM | 中 | ホテル詳細ページからの内部リンク | /deli/hotel/12345 から「同じ市区町村の他のホテル」「近くのラブホ」等への内部リンクがあると、Googleがホテル詳細をクロールする際の周辺発見性が上がる |
| MEDIUM | 易 | 「全国へ」ボタンのリンク先 | data-action="goToNationalTop" でJS経由で /deli/ に遷移するが、`<a href="/deli/">` でも同じ動作にすればGooglebotがリンクとして認識できる（現状はボタン要素なのでクロール対象外） |
| LOW | 易 | 主要都市リンクの深さ | sitemap には主要15都道府県の市区町村URLがあるが、ホテル一覧の都道府県ページに「主要市区町村バー」があると、Googlebotが市区町村ページに来やすい |

## 3. 用語解説（SEO初心者向け）

- **canonical（カノニカル）**: 「このページの正規URLはこちらです」とGoogleに教えるmetaタグ。同じ内容が複数URLにある時、SEOパワーを正規URLに集約させる
- **重複コンテンツ**: 同じ内容が複数のURLで配信されている状態。Googleが評価を分散したり、低品質と判断する原因
- **ロングテール**: 検索ボリュームは小さいが具体性が高いキーワード（例: "新宿 ビジネスホテル デリヘル カードキー"）
- **被リンク（バックリンク）**: 他サイトから自サイトへのリンク。SEOで最も重要な評価要素のひとつ
- **E-E-A-T**: Experience（経験）/ Expertise（専門性）/ Authoritativeness（権威性）/ Trustworthiness（信頼性）。Googleの品質評価軸
- **soft 404**: 200レスポンスを返しているが内容が空・薄いとGoogleに判断されたページ。インデックスから外される
- **CTR（Click-Through Rate）**: 検索結果に表示された回数のうち、クリックされた割合。title/description/構造化データで改善できる
- **ドメインオーソリティ**: ドメイン全体の評価値。被リンクの量と質で決まり、新規ページの順位上昇速度にも影響する
